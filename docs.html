Local Machine Setup

Cleanmails is optimized for local environments. This ensures your data never leaves your computer, providing 100% privacy for your email lists.

1Install Node.js

Before you begin, ensure you have Node.js installed on your system. We recommend version 18.0 or higher.

node --version

2Extract & Install

Unzip the Cleanmails package and navigate to the directory in your terminal.

npm install

3Run Development Server

Launch the dashboard locally to start cleaning lists.

npm run dev

Deploying to Vercel

Vercel is our suggested platform for production. It's fast, secure, and handles the build process automatically from your GitHub repository.

Why Vercel? Vercel's Edge Network ensures that your validator dashboard is responsive globally, and its free tier is more than enough for individual founders.

1GitHub Setup

Push your Cleanmails source code to a Private GitHub repository. Never make your validator public.

2Connect to Vercel

Visit vercel.com/new and select your private repository. Vercel will automatically detect the Vite configuration.

3Build Settings

Confirm the following settings:

  • Build Command: npm run build
  • Output Directory: dist

4Deploy

Click "Deploy". In less than 60 seconds, your validator will be live on a secure .vercel.app subdomain.

Deploying to Netlify

Netlify provides a robust platform with excellent build pipelines. It's an ideal alternative if you prefer their ecosystem over Vercel.

Steps to Deploy:

  1. Log in to Netlify and click Add new site.
  2. Connect your GitHub account and select your validator repository.
  3. Set the Build Command to npm run build and the Publish Directory to dist.
  4. Netlify will automatically trigger a build every time you push to your main branch.

Coolify & VPS Hosting

For users who want full control over their infrastructure, Coolify is a self-hosted PaaS that makes managing your own VPS as easy as managed services.

# Typical Coolify environment detection
Detected: Node.js / Vite
Nixpacks: Active

Process:

1. Connect your server to Coolify.
2. Create a new Resource and select "Public/Private Repository".
3. Coolify will use Nixpacks to build the project. No extra configuration is needed.
4. Ensure you set the base directory to ./ if your files are in the root.

Environment Variables

Cleanmails works out of the box without complex environment variables. However, if you're integrating with third-party tools, you can configure them here.

Batch Processing Tips

To ensure maximum accuracy without overwhelming your system:

  • Keep lists under 50,000 emails per batch.
  • Use a stable internet connection for DNS resolution.
  • Export results as CSV regularly to keep local state clean.
