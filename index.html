When you run an email through Cleanmails, it undergoes a deep "Multi-Layered Scan" to ensure it is safe for your sender reputation:

1. Strict Syntax Check

2. Real-time DNS Lookup

3. MX Record Verification

Specifically checks if the domain has "Mail Exchange" records configured to receive emails. An email with no MX records is 100% undeliverable.

4. Social Identity Match (Gravatar)

Performs a cryptographic hash check against Gravatar. If a profile exists, the email is marked as "High Confidence" because a real human has linked it to a social account.

5. Disposable/Burner Detection

Cross-references the domain against a database of 500+ temporary email providers (like Mailinator or 10MinuteMail) that are often used for fraud.

6. Role-Based Account Filtering

Identifies generic addresses like admin@, support@, or info@. These are "Risky" for outreach because they usually reach a group, not a decision-maker, and have higher complaint rates.

7. Bot & Entropy Analysis

Uses a custom algorithm to calculate the "randomness" of the local part. If an email looks like xjh23_9sb@, it is flagged as a likely robotic or machine-generated address.

8. Typo & Domain Reputation Check

  • Typo Detection: Checks for common mistakes (e.g., user@gmal.com) using Levenshtein distance.
  • Blacklist Check: Scans domain TXT records for security blocklists.
  • High-Risk TLDs: Flags domains using "cheap" extensions (like .top, .xyz, or .click) which are statistically more likely to be used for spam.
